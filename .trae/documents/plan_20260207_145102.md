## 解决方案

### 1. 修复底部导航栏问题
- 修改 `src/pages.json` 中的 tabBar 配置，确保格式正确
- 添加必要的配置项（如 iconSize、spacing 等）

### 2. 添加灵活的 Mock 模式切换功能
- **修改 `src/utils/constants.js`** - 添加 `USE_MOCK` 配置项（默认 true）
  ```js
  export const USE_MOCK = true  // true=使用Mock数据, false=使用真实接口
  ```

- **创建 `src/mock/index.js`** - Mock 数据主文件
  - 实现所有 API 的 mock 响应
  - 模拟网络延迟

- **创建 `src/mock/data.js`** - Mock 数据存储
  - 店铺详情数据
  - 商品列表数据
  - 订单列表数据
  - 用户数据等

- **修改 `src/api/index.js`** - 添加 Mock 模式切换逻辑
  - 导入 `USE_MOCK` 配置
  - 在 request 函数中判断是否使用 mock
  - 如果 USE_MOCK=true，返回 mock 数据
  - 如果 USE_MOCK=false，发送真实网络请求

### 3. 使用方式
只需修改 `src/utils/constants.js` 中的 `USE_MOCK` 值即可切换模式：
- `USE_MOCK = true` → 使用 Mock 数据（无需后端）
- `USE_MOCK = false` → 使用真实接口（需要后端服务）
