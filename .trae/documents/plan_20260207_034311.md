# 分支管理计划 - 简化版

## 📋 当前状态

* 当前分支: `main` (Web版本)

* 工作区: 干净，无未提交更改

## 🎯 分支策略

```
main          → 改造为微信小程序版本（主流应用方向）
feature/web    → Web版本备份（保留原生浏览器访问）
```

## 🚀 执行步骤

### 1. 基于main创建feature/web分支并推送到远程

```bash
git checkout -b feature/web
git push -u origin feature/web
```

### 2. 切回main分支准备小程序开发

```bash
git checkout main
```

## 📝 分支说明

* **main**: 将改造为微信小程序版本（主流应用方向）

* **feature/web**: Web版本完整备份（保留原生浏览器访问功能）

## ✅ 优势

* ✅ 操作简单，只需创建一个新分支

* ✅ Web版本完整保留在feature/web

* ✅ main直接改造为小程序，无需迁移

* ✅ 可随时切换回feature/web查看或恢复Web版本

## 🔄 后续工作流

* 小程序开发 → 在 `main` 分支进行

* Web版本维护 → 切换到 `feature/web` 分支

* 需要对比参考 → 在两个分支间切换查看
